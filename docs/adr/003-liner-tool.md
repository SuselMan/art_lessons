# ADR 003: Линер / Fineliner — чернильный инструмент рисования

## Статус

Accepted (2026-07-21). Определяет скоуп эпика «Линер/Fineliner» и его
саб-задач. Первая точка расширения тулсета за пределы графита — см. правку
`docs/MANIFESTO.md` (Vision / Принцип 3) и `docs/TOOLSET.md` от той же даты.

## Контекст

### Что такое линер физически

Одноразовая ручка с пористым/волоконным наконечником и пигментными чернилами.
Характер: линия почти постоянной ширины (нажим/наклон влияют слабо), край
слегка взаимодействует с зерном бумаги, при остановке — небольшое капиллярное
пропитывание, наконечник изнашивается со временем.

Главная метафора: **наконечник, пропитанный чернилами, касается бумаги
небольшой площадью и непрерывно отдаёт жидкость в её волокна** — не карандаш
(материал не накапливается поверх бесконечно, бумага впитывает чернила,
повторный проход почти не темнит уже насыщенную линию).

### Почему это не укладывается в текущий движок как есть

Весь pressure/tilt→геометрия даба код сейчас живёт в одном месте,
общем для всех инструментов: `DabSystem._makeDab`
(`apps/web/src/engine/src/DabSystem.ts`):

```ts
const size       = baseSize * (0.3 + 0.7 * pressure)         // ±70% от pressure
const aspectRatio = 1 + tiltNorm * tiltNorm * tiltNorm * 6.0  // до x7 от наклона
```

Это карандашные диапазоны — нажим и наклон должны сильно менять след. У
линера пресс/наклон должны влиять на 7–15%, не в разы (см. «Решение» ниже).
Сейчас per-tool ветвится только *opacity* (`_bakeDabOpacity`), не форма/размер
даба — значит для линера нужен предварительный рефакторинг: сделать
size/aspect-shaping конфигурируемым per-tool, а не только opacity.

Композитинг (`shaders.ts`, `gl_FragColor = vec4(u_color * deposit, deposit)`
+ alpha-over blend) уже насыщается к непрозрачности сам по себе — не
бесконечно темнеет при повторных проходах. Для линера это в целом подходит,
не требует новой схемы блендинга — только правильную кривую opacity/flow в
пресете (первый проход сразу почти насыщенный).

## Решение

### 1. Форма контакта

Круглый dab, лёгкая эллиптичность от наклона (короткий цилиндрический
наконечник — не длинный эллипс, как у карандаша при наклоне):

```
pressureExpansion = 0.92 + 0.12 * smoothPressure   // ширина: baseWidth * lerp(0.94, 1.08, pressureCurve)
aspect            = 1 + 0.15 * tiltNorm            // не tiltNorm³ * 6, как у карандаша
```

Для Technical Pen/Rapidograph-режима (follow-up, не в v1):
`pressureExpansion = 1`, `aspect = 1` почти всегда.

### 2. Нажим

Слабое влияние на ширину и на flow, никогда не `size = baseSize * pressure`
(это ощущалось бы как кисть/перо, не линер):

```
width = baseWidth * lerp(0.94, 1.08, pressureCurve)
flow  = baseFlow  * lerp(0.9, 1.12, pressureCurve)
```

### 3. Скорость

Влияет в основном на flow (жидкость на единицу длины), не на ширину:
`flowPerLength = baseFlow * speedResponse(speed) * pressureResponse(pressure)`,
где `speedResponse` — мягкий диапазон, ~1.08 на медленном движении до ~0.88
на очень быстром. Медленное движение/остановка — чуть плотнее и более
насыщенный край (см. dwell в п.6), быстрое — чуть светлее, зерно бумаги
заметнее.

### 4. Впитывание бумаги (halo)

Два визуальных компонента: **core** (прямой контакт, почти непрозрачный,
геометрически стабильный) и **wicking** (узкий мягкий ореол вокруг,
зависящий от типа бумаги, сильнее на медленном движении/остановке, почти
отсутствует на Bristol/smooth).

v1: `wick = edgeMask * paperAbsorbency * dwellFactor` — без биаса вдоль
направления волокна (это even cheaper approximation, выглядит убедительно
без реальной жидкостной симуляции). Настоящий fiber-direction bias
(`localFiberBias`, чтение локального направления волокна из уже существующей
paper-grain карты) — explicitly deferred, см. «Потом» ниже.

**Важно, cross-device determinism (`.claude/rules.md`):** если halo будет
модулироваться существующей paper-grain текстурой (общий холст, должен
рендериться идентично у всех участников комнаты), сам эффект обязан
оставаться чисто функцией уже испечённых, детерминированных данных (paper
grain map, позиция даба) — не вводить новую live per-device GPU float
математику, к которой применимо всё то же предупреждение, что уже трижды
ловило регрессии на paper grain (см. `feedback`/`project_paper_grain_redesign`
и сам файл правил).

### 5. Насыщающийся композитинг

`newInk = oldInk + deposit * (1 - oldInk)` — уже соответствует тому, как
устроен текущий accumulation-blend (см. «Контекст» выше). Для линера просто
нужна кривая opacity/flow в пресете, где первый проход уже даёт почти
насыщенный чёрный, а второй в основном закрывает микропропуски бумаги, а не
удваивает плотность центра.

### 6. Старт и конец линии

- Без pressure-taper до нуля (нижний предел ширины даже при почти нулевом
  pressure — `widthPressure = lerp(0.94, 1.08, pressure)`), иначе цифровой
  линер превращается в каллиграфическое перо на отрыве стилуса.
- Небольшая dwell-точка (чуть более тёмная/насыщенная) при остановке перед
  подъёмом — не выраженная клякса.
- Быстрый отрыв → короткое сужение на 5–15%, не до нуля.

### 7. Угол стилуса

`tilt < 55°`: почти без эффекта. `55–70°`: лёгкая эллиптичность (см. п.1).
`70°+`: небольшое снижение flow, возможны микропропуски — эффект
экстремального диапазона, не агрессивный на обычных углах письма.

### 8. Фактура и микропропуски

`coverage = contactMask * lerp(0.96, 1.0, paperContact) * flow` — бумага
влияет заметно слабее, чем на графит (`ANCHOR_*` кривые в
`pencilPresets.ts` для сравнения масштаба). На Bristol — почти печатная
линия, на rough paper — чуть живее, редкие микроскопические светлые поры,
которые заполняются при повторном проходе.

### 9. Пересечения и повторные проходы

Без wetness/времени высыхания в v1 — насыщающегося композитинга (п.5)
достаточно: центр пересечения почти не темнеет вдвое, край может слегка
расшириться из-за дополнительной жидкости.

## MVP-скоуп (Liner v1)

- Форма: почти круглый dab, ширина ±7% от pressure, наклон даёт до 10–15%
  эллиптичности (п.1–2).
- Подача: почти постоянная, быстрое движение — до ~12% светлее (п.3),
  saturating accumulation (п.5).
- Бумага: слабые микропоры внутри, фактура в основном на краях, небольшой
  absorb halo без волоконного bias (п.4, п.8).
- Начало/конец: без pressure taper в ноль, маленькая dwell-точка на
  остановке (п.6).
- Наклон: п.7, без агрессивного поведения на обычных углах.
- Размеры: 0.1, 0.2, 0.3, 0.5, 0.8 мм — фиксированные шаги, свободный размер
  в advanced settings.
- Цвет: произвольный (переиспользование `ColorPicker`/`StrokeOperation.color`,
  как у Color pencil — см. `docs/TOOLSET.md`, «Пересмотр: два
  pencil-инструмента»), не только чёрный.
- Сглаживание траектории: переиспользовать текущую Catmull-Rom/corner-preserving
  систему `DabSystem` как есть — **не** добавлять искусственный jitter поверх
  хорошего стилуса; линия должна остаться чистой, но не выглядеть как
  vector-simplify-path.

## Потом (explicitly out of v1 epic, отдельные issues)

- Wicking вдоль направления волокна бумаги (`localFiberBias` из paper-grain
  карты, п.4) вместо простого halo.
- Износ наконечника (постепенное «размягчение» строгости линии).
- Отдельный Technical Pen / Rapidograph пресет (см. п.1, п.7) с фикс.
  калиброванными ступенями ширины 0.1–1.0мм.
- Wetness/время высыхания для более точного поведения при пересечении
  свежих линий (п.9).
- Звук инструмента (по аналогии с `PencilSound.ts`).

## Явно вне скоупа этого эпика

- Manga/G-pen (нажимное перо с сильной pressure→width кривой) и
  perspective/ellipse guide — обсуждались как соседние кандидаты в тулсет,
  сознательно не заводятся сейчас (см. `docs/TOOLSET.md`).
- Screentone — конфликтует с текущей анти-целью «без fill bucket», отдельное
  продуктовое решение, не бандлится с линером.
